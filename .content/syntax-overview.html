psl syntax

tokens:

left - holds the value of false
right - holds the value of true

operators:

like - equals comparator
+, -, /, *, <, >

1 + 1 = 2
1 - 1 = 0
1 * 1 = 1
1 / 1 = 1
1 < 2 = true (1 is less than 2)
1 > 2 = false (1 is not greater than 2)

can.you.not( ) - not operator (Makes an operator do the opposite of what it should)

# - required to reference variables. You cannot reference a variable that has not been declared using a type

primitive types:

uggs - booleans (holds either left or right) a statement that is true or false
short - short
tall - int
grande - float
venti - long
trenta - double
retweet - strings (it’s like a word or multiple words)

scopes:

um - if
An if statement is used to check if a condition is true or false

duh - end if

so - while
a while loop is a segment of code that will continue to happen until a condition is met.
that means the code will continue until something inside the loop causes the condition to be false.

totes - end loop

identifiers:

Identifiers are variable names.
Names can be made up of (like how you would name a hashtag):
upper-case letters
lower-case letters
numbers

No special characters

i/o:

can.i.just.say.something:- stdout
This is what people will be seeing. It is what will be displayed on the screen

i.cant.even: - stderr
This is what will be displayed if something is wrong

newsfeed: - stdin
This is what you will be typing into your program

Syntax/Grammar Rules:

White Spaces - there must be at least one whitespace between tokens, but they don’t have any special meaning
Program must contain the following open and close tags

Variable declaration:
Variables must be declared with a type as its type modifier

Scopes: must be followed by a conditional and a colon
