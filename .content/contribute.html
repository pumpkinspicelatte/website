contributing to psl

report issues

issues or bugs are problems with the output of the code that you write in psl, like the output doesn't match your logic.

an example of a bug that was found:
short ana = 0

so #ana < 10:
#ana = #ana + 1
totes

can.i.just.say.something: #ana

logically, this should output: 10
but, the output read: 2

this was due to an error in the comparison during evaluation, and was fixed.

in order for a bug to be fixed, it needs to be reported, which can be done by visiting psl's git repository where the source is housed.

to report an issue: click here

request features

a feature would be a suggesiton for something to be added into the website or language.

if you visit the git repo, you'll find, at the bottom of the README, a section called "to be added" which are features that will be added in the future.

to request a feature: click here
these will be requested the same way issues are reported.

add to psl source

if you want to become a contributor to the source for psl, you'll have to create an account on GitLab and go to the psl repo to request access.

you can also email us at: contribute.to.psl@gmail.com

with your GitLab username, and a resume/portfolio (so we know you're serious about contributing).
